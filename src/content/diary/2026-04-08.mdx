## 日记页 SSR 修复

- 根本原因：日记页原为纯客户端组件，useEffect fetch /api/diaries 在生产环境静默失败，导致永远卡在 loading 状态
- 修复方案：改为 SSR（Server Component），数据在服务端直接通过 fs 读取 MDX，HTML 输出时已包含完整内容
- 改动：新建 DiaryClient.tsx（交互层：标签筛选），page.tsx 改为 async Server Component（数据层）
- 同步修复：笔记页无此问题（数据是静态 import，不依赖 fetch）

## 新自动化｜一锤日记同步到博客（17:29）

大爷命名：一锤 = 本地小锤子（OpenClaw workspace），二锤 = WorkBuddy 侧（我）

**新增自动化**：每天 7:30 拉取一锤 Apple 备忘录"🔨 小锤子"文件夹中的日记 → 转换为 MDX → 推送到博客 GitHub → Vercel 自动部署。

**链路**：
1. AppleScript 读取 Apple 备忘录"🔨 小锤子"中标题为"小锤子日记｜YYYY-MM-DD"的笔记
2. Python 解析 HTML 内容，提取"今天一句话/关键决定/实际进展/碎碎念/回看/瞬间"等章节
3. 构建 MDX 文件（含 frontmatter：title/date/excerpt/tags）
4. 写入 blog/src/content/diary/YYYY-MM-DD.mdx
5. Git SSH 推送到 GitHub，触发 Vercel 部署

**脚本**：`/Users/kdnsna/WorkBuddy/Claw/.workbuddy/scripts/sync-yizhui-diary-to-blog.py`

**自动化**：macOS LaunchAgent
- Label: `ai.kdnsna.workbuddy.diary-sync`
- 路径：`~/Library/LaunchAgents/ai.kdnsna.workbuddy.diary-sync.plist`
- 触发时间：每天 07:30
- 日志：`~/.workbuddy/logs/diary-sync.log`

**测试**：2026-04-07 已成功跑通，MDX 生成正确，GitHub push 成功。
