## 数据层 (src/lib/)

### 核心工具

| 文件 | 说明 |
|------|------|
| diary.ts | **日记数据获取** - 读取 MDX 文件，解析 frontmatter |
| notes.ts | **笔记数据** - 静态笔记数据（hardcoded） |
| home.ts | **首页数据聚合** - 组合门户、更新、精选等数据 |
| story.ts | **故事页数据** - 时间线、日记分组逻辑 |
| method.ts | **方法页数据** - 方法笔记分类 |
| achievement.ts | **成果页数据** - 项目、案例、日志 |
| types.ts | **类型定义** - 统一的 TypeScript 接口 |
| supabase.ts | **Supabase 客户端** - 数据库连接 |
| sensitive-mask.ts | **敏感信息脱敏** - 日记发布前脱敏处理 |
| auto-reply.ts | **自动回复** - 茶话会自动回复逻辑 |
| musings.ts | **碎碎念数据** |

### 关键数据结构

```typescript
// 日记条目
interface DiaryEntry {
  slug: string          // 文件名（日期）
  date: string          // 日期
  title: string         // 标题
  excerpt: string       // 摘要（脱敏）
  tags: string[]        // 标签
  content: string       // 正文（脱敏后的HTML）
}

// 笔记条目
interface Note {
  id: string
  category: string
  categoryColor: string
  icon: string
  title: string
  description: string
  detail: string        // 完整内容（Markdown）
  tags: string[]
  date: string
  featured?: boolean
}
```

## 内容管理

### 日记同步流程（核心！）

```
定时任务 (crontab)
    ↓
scripts/sync-diary.ts
    ↓
读取 ~/.workbuddy/memory/*.md
    ↓
解析内容 + 敏感信息脱敏
    ↓
生成 MDX 格式
    ↓
写入 src/content/diary/YYYY-MM-DD.mdx
    ↓
GitHub → Vercel 自动部署
```

### 日记文件格式

```mdx
---
title: "日记标题"
date: "2026-04-07"
excerpt: "摘要内容..."
tags: ["标签1", "标签2"]
---

日记正文内容...

## 二级标题

正文...
```

### 笔记数据（硬编码）

笔记数据存储在 src/lib/notes.ts 中，是静态的 JavaScript 数组

## 扣子编程升级指南

### 读取文件
使用 GitHub API 或本地克隆后读取

### 修改文件
1. 读取原文件内容
2. 修改后提交到 GitHub
3. Vercel 自动检测并部署

### 注意事项
- **日记内容**：src/content/diary/ 目录，由同步脚本自动生成
- **笔记内容**：src/lib/notes.ts 硬编码数组
- **样式修改**：优先修改 CSS Modules，保持组件独立
- **敏感信息**：发布前必须通过 sensitive-mask.ts 脱敏
